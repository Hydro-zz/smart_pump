# Smart Pump - Система управления дренажным насосом

Полностью автоматическая система управления дренажным насосом на базе **ESP8266** с веб-интерфейсом и поддержкой Home Assistant.

## 🎯 Основные возможности

- **Автоматическое управление** - включение/выключение по уровню давления
- **Веб-интерфейс** - управление и мониторинг в реальном времени
- **Ручной режим** - полный контроль с кнопок на интерфейсе
- **OTA обновление** - обновление прошивки по WiFi с аутентификацией
- **Защитный таймаут** - автоматическое отключение при превышении времени работы
- **JSON API** - интеграция с Home Assistant и другими системами
- **Режим конфигурации** - быстрая настройка параметров через AP точку доступа
- **Мониторинг WiFi** - отображение уровня сигнала и качества подключения

## 🔧 Характеристики

| Параметр | Значение |
|----------|----------|
| **Микроконтроллер** | ESP8266 |
| **Датчик давления** | TM7711 (24-бит) |
| **Реле** | На GPIO 5 (D1) |
| **Кнопка настройки** | GPIO 0 (D3) |
| **Интерфейс датчика** | SPI (SCK на D5, DT на D6) |
| **Частота обновления** | 500 мс |
| **Портал управления** | HTTP на порту 80 |

## 📦 Требуемые библиотеки

Установите через Arduino IDE:
```
- ESP8266WiFi
- ESP8266WebServer
- ESP8266HTTPUpdateServer
- EEPROM (встроенная)
```

## ⚙️ Подключение компонентов

```
ESP8266 Pin      → Компонент
GPIO 5 (D1)      → Реле (управление насосом)
GPIO 0 (D3)      → Кнопка "Setup" (pull-up)
GPIO 14 (D5)     → TM7711 Clock (SCK)
GPIO 12 (D6)     → TM7711 Data (DT)
GND              → GND всех компонентов
3.3V             → VCC датчика и реле
```

## 🚀 Быстрый старт

### 1. Первая загрузка

1. Подключите ESP8266 к компьютеру через USB
2. Откройте `smart_pump.ino` в Arduino IDE
3. Выберите правильную плату и COM порт
4. Загрузите скетч

### 2. Начальная конфигурация

При первом запуске устройство будет работать в **режиме конфигурации (AP)**:

1. Найдите WiFi сеть **`Smart_pump`** на вашем устройстве
2. Подключитесь к ней (пароль не требуется)
3. Откройте в браузере: `http://192.168.4.1`
4. Заполните параметры:
   - **SSID и пароль WiFi** - для подключения к домашней сети
   - **Логин/пароль администратора** - для защиты страницы обновления
   - **Нижний порог давления** - при достижении выключается насос
   - **Верхний порог давления** - при превышении включается насос
   - **Таймаут (сек)** - максимальное время работы насоса
5. Нажмите "Сохранить все настройки"
6. Устройство перезагрузится и перейдет в рабочий режим

### 3. Работа в нормальном режиме

- **Адрес в сети**: Найдите IP адрес ESP8266 в настройках маршрутизатора
- **Доступ к интерфейсу**: `http://<IP_адрес>`
- **JSON API**: `http://<IP_адрес>/api/status`

## 🎮 Режимы работы

### Авто-режим (по умолчанию)
- Насос автоматически включается при превышении верхнего порога давления
- Автоматически выключается при понижении ниже нижнего порога
- При срабатывании таймаута насос блокируется на текущей сессии

### Ручной режим
- Нажмите кнопку "Перейти в РУЧНОЙ режим"
- Управляйте насосом кнопками "ВКЛЮЧИТЬ" / "ВЫКЛЮЧИТЬ"
- Вернитесь в авто-режим кнопкой "Вернуть АВТО режим"

## 📊 Веб-интерфейс

### Главная страница
- **Индикатор уровня давления** - визуальная полоса в процентах
- **Статус системы** - текущее состояние (авто/ручной, включен/выключен)
- **Таймер работы** - время, прошедшее с момента включения
- **Время до отключения** - оставшееся время на основе таймаута
- **Управление режимами** - переключение между авто и ручным режимом
- **Параметры давления** - редактирование пороговых значений
- **Информация WiFi** - качество сигнала в процентах и dBm

### JSON API (`/api/status`)

```json
{
  "pressure": 5400,
  "percent": 45,
  "pump_state": true,
  "manual_mode": false,
  "timeout_triggered": false,
  "low_level": 2000,
  "high_level": 8000,
  "timeout_value": 60,
  "running_seconds": 25,
  "seconds_left": 35,
  "wifi_rssi": -65,
  "wifi_pct": 78
}
```

## 🔐 Безопасность

- **OTA обновления защищены** - требуется логин/пароль администратора
- **OTA поля пусты при старте** - первый запуск всегда требует конфигурации
- **Таймаут включения** - защита от зависания насоса в сломанном состоянии
- **Проверка целостности памяти** - контрольная сумма конфигурации (configID)

## 🔄 OTA Обновление

1. Скомпилируйте новую версию в Arduino IDE
2. Выберите `Sketch → Export Compiled Binary` (получите `.bin` файл)
3. Откройте `http://<IP_адрес>/update` в браузере
4. Введите логин/пароль администратора
5. Выберите `.bin` файл и нажмите "Запустить update"
6. Ждите перезагрузки устройства (~30 сек)

## 🏠 Интеграция с Home Assistant

Используйте JSON API для создания сенсоров и переключателей в Home Assistant:

```yaml
# configuration.yaml
sensor:
  - platform: rest
    resource: http://<IP_АДРЕС>/api/status
    name: "Давление колодца"
    json_attributes:
      - pump_state
      - timeout_triggered
      - wifi_pct
    unit_of_measurement: "мбар"
    value_template: "{{ value_json.pressure }}"
    scan_interval: 10
```

## 🐛 Решение проблем

### Устройство не найдено в сети
- Проверьте правильность SSID и пароля
- Перезагрузите маршрутизатор
- Нажмите кнопку "Setup" на ESP8266 и переконфигурируйте

### Датчик показывает странные значения
- Убедитесь в надежности подключения проводов SPI
- Перезагрузите устройство
- Проверьте калибровку датчика TM7711

### OTA обновление не работает
- Убедитесь, что установлены логин и пароль (конфигурация)
- Проверьте размер `.bin` файла
- Имеется ли достаточно памяти на чипе
